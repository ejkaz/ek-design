Thesis · Dark Ledger / Neon Operator

Principle · v0.2.1

Purple is the world. Magenta is the command. Cyan is the proof.

Static finance truth is sober. Agentic execution is luminous.
Source proof is cyan. Attention is amber. Risk is red.

Violet-black is the operating environment.
Steel-violet is the inactive architecture: borders, nested panels,
dormant machine states, secondary chrome.
Magenta means agentic command, automation, override, or approval.
Cyan means traceability, source lineage, drill-through, sync proof,
terminal output, and inspectability.
Lime means health, completion, successful automation, or live status only.
Amber means watch, stale, uncertain, approval required, or CFO attention.
Red means true risk, failed sync, covenant breach, runway breach,
material error, or destructive action.

Layer separation: the ledger layer is dark, sober, and numerically precise · the architecture layer is steel-violet · the command layer is magenta · the proof layer is cyan · the attention layer is amber · the health layer is lime · the risk layer is red.

Identity

Operator-facing software register pulled from the visual vocabulary of Neo-Tokyo cinema (Blade Runner 2049, Ghost in the Shell) and real Tokyo-after-dark neon, tightened around an agentic finance cockpit thesis: "Dark Ledger / Neon Operator." The static finance truth layer is dark and sober. The data lineage layer is cyan. The agentic execution layer is magenta. The CFO attention layer is amber. The health/reconciliation layer is lime. The risk layer is red. Steel-violet is atmospheric depth only — rarely text. Built for two concurrent purposes: 1. Obsidian hub for project management (cepsra side) 2. FinanceOS — agentic finance control plane connecting QuickBooks, forecasting, burn analysis, headcount, AWS/AI unit economics JetBrains Mono carries every identifier (model name, agent ID, timestamp, file path, token count, hash, period, source freshness). Tabular numerals carry every finance number. Eric is always the operator; the UI is always one of his agents.

Major changes from v0.1.0:
  - SURFACES: background undertone shifted from `#0A0A14` (pure
    blue-black) to `#070711` (deeper violet-black). Layered surfaces
    also re-tuned: panel `#15151F` → `#171727` (surface_2), with new
    surface_1 `#10101A` and surface_3 `#23233A` slots. Border default
    `#2E2E40` → `#2C2A45` (steel-violet-tinted). Background reads as
    "finance ledger black with violet atmospheric undertone."
  - PRIMITIVES: added `finance:` family — semantic directionality
    colors distinct from agent state. gain (#4ADE80) / loss (#F87171)
    / watch (#FBBF24) / info_blue (#60A5FA). Used only for variance,
    delta, financial health — never as brand accent or chart color
    when the series isn't financially-directional.
  - ROLES: added 4 new role groups —
      `agent.*` (command, trace, attention, complete, risk, idle) —
        tightens v0.1's agent_status_* with stronger semantic locks
      `finance.*` (cash, runway, burn, arr, gross_margin,
        forecast_variance, budget_variance, headcount_plan,
        aws_ai_cogs) — semantic matrix for FinanceOS surfaces
      `data.*` (live, synced, stale, failed, manual_override) —
        data freshness / sync state
      `trace.*` (rail, source, breadcrumb, drill_through) — cyan
        lineage / inspectability
  - MOTIFS: added 8 finance-oriented motifs (dark-ledger-panel,
    neon-operator-layer, trace-rail, agent-event-feed,
    forecast-confidence-halo, board-safe-mode, executive-command-state,

  - TYPOGRAPHY: added `modes` block documenting two contexts —
    brand_portal (Chakra Petch headers + Inter body) and
