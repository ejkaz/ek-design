Surfaces — mode per surface

App

mode: dark · background: {neutral.950} → #070711
Examples: app-shell, dashboard, agent-console, task-list, briefing-view, chat-panel

cepsra app surface defaults to dark. Reduces eye strain during long operator sessions; neon accents read vividly on near-black; mono identifiers maintain phosphor-CRT legibility. All major agentic UI references (Claude desktop, ChatGPT desktop, Cursor, Linear dark) are dark-default.

Cockpit

mode: dark · background: {neutral.950} → #070711
Examples: finance-cockpit, fp&a-dashboard, runway-view, burn-analysis, headcount-plan, board-pack, forecast-vs-actuals

FinanceOS surfaces. Same dark register as `app` but density expectation is higher — KPI strips, dense tables, source-trace rails, agent event feeds. Inherits `dark-ledger-panel` motif as default chrome. Cockpit content is sober; agentic elements (commands, approvals, automation) are the only places that emit `neon-operator-layer` glow.

Marketing

mode: dark · background: {neutral.950} → #070711
Examples: landing-page, hero-sections, feature-grid, pricing, docs-homepage

Marketing site sits in the same dark register as the app so brand-to-app transition is seamless. Cyberpunk register reads as "for operators" — self-selects the right audience.

Board Safe

mode: dark · background: {neutral.950} → #070711
Examples: pdf-export, board-screenshot, investor-update-export, partner-deck-asset

Same dark system but reduced neon — no scanlines, no decorative glow, focus-and-state glow only. Suitable for PDF exports and board screenshots where the cyberpunk register should sit beneath the data, not on top. Activated by a `.board-safe` body class that overrides effects.* tokens (see effects.board_safe_*).

Light

mode: light · background: {neutral.50} → #F0F0F5
Examples: print-views, accessibility-mode

Opt-in inverted mode for print and accessibility override. Rare. Same primary axis; only mode bindings differ.

Terminal Inset

mode: dark · background: {neutral.1000} → #000000
Examples: code-block, agent-log, command-output, jsonl-stream, token-debug, model-diff, sync-trace

The "console window inside the app" wedge. Pure-black ground + cyan border + JetBrains Mono content. Agentic UIs need a visually- distinct surface for raw machine output (logs, tool calls, JSON streams, sync traces, model diffs) — this is it. Use sparingly; not every dense surface needs terminal styling.

Invariants — hard rules

Surfaced from invariants. ek-craft enforces.

magenta_cta_monopoly
Magenta #FF2A6D (magenta.500) is the only primary-CTA color. No secondary may be used as a primary CTA fill.
Preserves the action signal across every ek surface. If cyan becomes a CTA too, the operator can't pre-attentively spot the action. error
cyan_secondary_and_info
Cyan #00D1FF (cyan.500) is restricted to secondary action, info, link, trace, terminal_inset text, and AA-bound chart series. NEVER a primary CTA fill.
Cyan competes hard with magenta for attention; reserving it for navigation (links), information (info badges), and lineage (trace) keeps the magenta CTA monopoly readable. error
lime_alive_and_success_only
Lime #A6FF00 (lime.500) appears ONLY in status_success_*, agent_status_active, agent_complete, finance_cash, finance_arr, data_live, gain roles. Never decorative.
Lime carries multi-bound semantic load (success + alive + health + gain). If it leaks into decoration, the operator can no longer trust 'green = working / healthy / favorable.' error
red_error_only
Red #FF003C (red.500) appears ONLY in status_error_*, destructive_action, agent_error, agent_risk, data_failed, finance loss/breach roles. Never decorative, never in chart series.
Semantic-locked statuses are non-negotiable. Red on this brand means 'something materially broke or is at material risk.' error
amber_warn_only
Amber #FFB800 (amber.500) appears ONLY in status_warn_*, agent_blocked, agent_attention, agent_approval_required, data_stale, finance_runway / forecast_variance roles.
Amber = CFO attention / watch state. Decorative amber dilutes the watch signal and trains the operator to ignore real attention prompts. error
mono_for_identifiers
JetBrains Mono is required for code, file paths, IDs, timestamps, model names, hashes, token counts, period codes, source-system names, and any other machine-generated identifier.
Agentic + finance UI is operator-facing; the operator scans identifiers constantly. Mono preserves alignment, prevents I/l/1 confusion, and visually separates 'machine output' from 'product chrome.' Single biggest UX win in operator interfaces. error
no_consumer_indigo_purple
Tailwind-indigo #6366F1 and consumer-violet #8B5CF6 / #A78BFA are forbidden. ek's atmospheric purple-blue is steel_violet.500 #3B3B92 (darker, more blue-violet).
Indigo-500 is the single most-overused color in 2024-2026 AI products. Defaulting to it instantly reads as 'AI-template starter' rather than brand. error
dark_primary_register
Surfaces app + cockpit + marketing default to dark mode (background #070711). Light mode is opt-in only (print, accessibility override).
The neon palette reads correctly on near-black; on light it's anemic. Dark-default is also the agentic-UI norm and the finance-cockpit norm (Bloomberg-style). warning
reduced_motion_required
Any animation (transitions, pulses, scanline overlay) must include an @media (prefers-reduced-motion: reduce) override that disables or substantially reduces motion.
Accessibility baseline. Scanline overlay especially can trigger vestibular issues. error
finance_directionality_locked
The finance.* primitives (gain #4ADE80, loss #F87171, watch #FBBF24, info_blue #60A5FA) appear ONLY in variance, delta, financial directionality, and data freshness contexts. Never as brand accent, never as decorative chart color, never as a button background.
Dark Ledger thesis: finance directionality is a semantic load-bearing signal. Operator scanning a P&L must trust that green = favorable, red = unfavorable, amber = watch — without ambiguity from decorative reuse. Adopted from Bloomberg's variance-color discipline. error
agent_command_magenta_only
Magenta is the agentic command / action / automation color. Used in CTAs, agent_command_active state, executive command bars, approval-required affordances, manual-override indicators. NEVER as nightclub-decorative neon, NEVER on a chart series that isn't agentic, NEVER as a brand-accent pink.
Reframes v0.1's CTA monopoly with a sharper semantic: magenta = 'an agent is doing something or you are commanding one.' If magenta means anything else (chart fill, decoration, hover-glow-on-passive-card), the operator can't trust the command signal. error
cyan_for_trace_only
Cyan is the inspectability / trace / lineage / drill-through color. Used for source labels, trace rails, drill-through arrows, info, link, terminal output. NEVER as a primary CTA, NEVER as a decorative accent, NEVER as a brand-accent neon.
Dark Ledger thesis: cyan answers 'where did this number come from?' If cyan also means CTA or accent, the operator can't trust the inspectability signal. Cyan is the 'show your work' color. error
tabular_numerals_for_finance
All finance numerics (KPI values, table cells, variance %, delta, currency amounts, percentages) must use JetBrains Mono with `font-feature-settings: "tnum"` (or equivalent OpenType `tnum` flag). Proportional numerals are forbidden in finance contexts.
Finance data is scanned in columns. Proportional numerals misalign across rows and across rows-with-different-magnitudes, which makes variance scanning slower and bug-prone. Tabular numerals lock column alignment. Bloomberg, FactSet, every serious finance UI uses tabular. error
no_em_dashes_in_copy
Em-dashes (— U+2014) are forbidden in brand copy, UI copy, and component labels. Use colon, parentheses, comma, or n-dash (–) where appropriate.
Em-dashes are a 2024-LLM-output tell. ek voice is operator-respecting and terse; em-dashes signal generated prose. n-dashes for numeric ranges remain fine. info
board_safe_export_mode
When the body has class `board-safe` (PDF export, board screenshot, investor-deck asset), all effects.neon_glow_* must collapse to effects.board_safe_neon_glow_* (inset 1px border, no outer glow), all scanline overlays must be removed, and all animations must be paused. Focus-ring stays visible for accessibility.
Board materials should let the data lead. Neon glow that's a feature in the live cockpit becomes noise in a PDF screenshot embedded in an IC memo. Board-safe mode is the export-pressure-valve so the cyberpunk register doesn't need to be diluted for live ops. warning

Avoid · forbidden colors / fonts / patterns

Surfaced from avoid. ek-craft lints any .tsx/.css against this list.

Generic AI-default palette — every Tailwind starter app uses this. The single most-overused color in 2024-2026 AI products. ek's steel_violet ramp is the differentiated atmospheric purple-blue.
Tailwind Violet 500 #8B5CF6 error
Same family as #6366F1 — consumer-SaaS-purple cliché.
Tailwind Violet 400 #A78BFA error
Pastel-violet — wrong register for cyberpunk + AI-default.
trust-gradient error
Purple-to-pink gradient backgrounds (#6366F1 → #EC4899 and similar).
Quintessential AI-slop visual. Reads as marketing-template-purchased.
pastel-tints warning
Any color with saturation < 30% AND lightness > 75%.
Pastels are off-register for cyberpunk. ek uses saturated neons on near-black.
Pure Cyan (#00FFFF) #00FFFF warning
Sci-fi-cliché cyan. ek uses #00D1FF (cyan.500) — biased slightly toward blue for a more refined, less Tron-cosplay register.

font: Arial error

font: Helvetica error
Reads corporate / generic. Inter + Chakra Petch are the locked stack.
font: system-ui error
Inherits OS defaults — never use as display.
font: Comic Sans MS error
Obvious.
font: Orbitron warning

pattern: glassmorphism-heavy-blur warning

pattern: gradient-mesh-background warning
Iridescent / mesh-gradient backgrounds read as 2022 design template, not cyberpunk.
pattern: emoji-as-icon error
Lucide SVG icons only. Emojis in UI chrome read as casual / consumer.
pattern: lorem-ipsum warning
Use real content or honest placeholders.
pattern: rounded-card-with-colored-left-border warning
Overused B2B SaaS trope.
pattern: decorative-neon-glow warning
Neon edge glow is reserved for focus, hover, and the agent_status_active badge. Decorative glow everywhere = cyberpunk costume, not brand.
pattern: decorative-glow-on-every-card warning
v0.2 Dark Ledger thesis: static finance panels stay sober (no glow by default). Only agentic elements (commands, active automation) emit neon-operator-layer glow. Glow on every card collapses the operator-vs-truth distinction.
pattern: full-screen-purple-pink-gradient error
Synthwave / generic-AI tell. ek backgrounds are flat near-black with optional steel-violet atmospheric tint.
pattern: chromatic-aberration warning
