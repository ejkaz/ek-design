Surfaces — mode per surface

App

mode: dark · background: {neutral.950} → #0A0A14
Examples: app-shell, dashboard, agent-console, task-list, briefing-view, chat-panel

cepsra and any future agentic surface defaults to dark. Reduces eye strain during long operator sessions; neon accents read vividly on near-black; mono identifiers maintain phosphor-CRT legibility. All major agentic UI references (Claude desktop, ChatGPT desktop, Cursor, Linear dark) are dark-default.

Marketing

mode: dark · background: {neutral.950} → #0A0A14
Examples: landing-page, hero-sections, feature-grid, pricing, docs-homepage

Marketing site sits in the same dark register as the app so brand-to-app transition is seamless. Cyberpunk register reads as "for operators" — self-selects the right audience.

Light

mode: light · background: {neutral.50} → #F0F0F5
Examples: print-views, exported-pdf, accessibility-mode

Opt-in inverted mode for print exports, accessibility override, and any surface where dark mode is wrong (rare). Same primary axis; only mode bindings differ.

Terminal Inset

mode: dark · background: {neutral.1000} → #000000
Examples: code-block, agent-log, command-output, jsonl-stream, token-debug

The "console window inside the app" wedge. Pure black ground + cyan border + JetBrains Mono content. Agentic UIs need a visually-distinct surface for raw machine output (logs, tool calls, JSON streams) — this is it.

Invariants — hard rules

Surfaced from invariants. ek-craft enforces.

magenta_cta_monopoly
Magenta #FF2A6D (magenta.500) is the only primary-CTA color. No secondary may be used as a primary CTA fill.
Preserves the action signal across every ek surface. If cyan becomes a CTA too, the user can't pre-attentively spot the action. error
cyan_secondary_and_info
Cyan #00D1FF (cyan.500) is restricted to secondary action, info, link, terminal_inset text, and AA-bound chart series. NEVER a primary CTA fill.
Cyan competes hard with magenta for attention; reserving it for navigation (links) and information (info badges) keeps the magenta CTA monopoly readable. error
lime_alive_and_success_only
Lime #A6FF00 (lime.500) appears ONLY in status_success_*, agent_status_active, agent_status_complete roles. Never decorative, never in chart series without success semantics.
Lime carries dual semantic load (success + alive). If it leaks into decoration, the operator can no longer trust 'green = working'. error
red_error_only
Red #FF003C (red.500) appears ONLY in status_error_*, destructive_action, agent_error roles. Never decorative, never in chart series.
Same construction as magenta CTA monopoly — semantic-locked statuses are non-negotiable. error
amber_warn_only
Amber #FFB800 (amber.500) appears ONLY in status_warn_*, agent_blocked roles.
Locks the warn semantic; prevents amber-as-decorative drift. error
mono_for_identifiers
JetBrains Mono is required for code, file paths, IDs, timestamps, model names, hashes, token counts, and any other machine-generated identifier. Inter/Chakra Petch must NOT be used for these.
Agentic UI is operator-facing; the operator scans identifiers constantly. Mono preserves alignment, prevents I/l/1 confusion, and visually separates 'machine output' from 'product chrome'. Single biggest UX win in operator interfaces. error
no_consumer_indigo_purple
Tailwind-indigo #6366F1 and consumer-violet #8B5CF6 / #A78BFA are forbidden. ek-design's atmospheric purple-blue is steel_violet.500 #3B3B92 (darker, more blue-violet).
Indigo-500 is the single most-overused color in 2024-2026 AI products. Defaulting to it instantly reads as 'AI-template starter' rather than brand. error
dark_primary_register
Surfaces app + marketing default to dark mode (background #0A0A14). Light mode is opt-in only (print, accessibility override).
The neon palette reads correctly on near-black; on light it's anemic. Dark-default is also the agentic-UI norm (Claude desktop, Cursor, Linear dark, Vercel dark). warning
reduced_motion_required
Any animation (transitions, pulses, scanline overlay) must include an @media (prefers-reduced-motion: reduce) override that disables or substantially reduces motion.
Accessibility baseline. Scanline overlay especially can trigger vestibular issues — must be disable-able. error

Avoid · forbidden colors / fonts / patterns

Surfaced from avoid. ek-craft lints any .tsx/.css against this list.

Tailwind Indigo 500 #6366F1 error
Generic AI-default palette — every Tailwind starter app uses this. The single most-overused color in 2024-2026 AI products. ek-design's steel_violet ramp is the differentiated atmospheric purple-blue.
Tailwind Violet 500 #8B5CF6 error
Same family as #6366F1 — consumer-SaaS-purple cliché.
Tailwind Violet 400 #A78BFA error
Pastel-violet — wrong register for cyberpunk + AI-default.
trust-gradient error
Purple-to-pink gradient backgrounds (#6366F1 → #EC4899 and similar).
Quintessential AI-slop visual. Reads as marketing-template-purchased.
pastel-tints warning
Any color with saturation < 30% AND lightness > 75%.
Pastels are off-register for cyberpunk. ek-design uses saturated neons on near-black; pastels collapse the whole aesthetic.
Pure Cyan (#00FFFF) #00FFFF warning
Sci-fi-cliché cyan. ek-design uses #00D1FF (cyan.500) — biased slightly toward blue for a more refined, less Tron-cosplay register.
font: Roboto error
Generic Material-default font. Inter is the locked ek body face; Chakra Petch the display.
font: Arial error
System default — never use as display.
font: Helvetica error
Reads corporate / generic. Inter + Chakra Petch are the locked stack.
font: system-ui error
Inherits OS defaults — never use as display.
font: Comic Sans MS error
Obvious.
font: Orbitron warning
Sci-fi cosplay font — too costume-y for production agentic UI. Chakra Petch reads cyber-coded without going full Tron.
font: Press Start 2P warning
8-bit pixel cosplay — wrong register for agentic productivity.
pattern: drop-shadow-as-elevation warning
Cyberpunk elevation = neon edge glow (box-shadow with brand color), not gray drop shadow. See effects.neon_glow_*.
pattern: glassmorphism-heavy-blur warning
Heavy backdrop-blur reads as 2020-Apple, not cyberpunk. If you need a frosted surface, use a 4-8px blur max plus subtle border-glow.
pattern: gradient-mesh-background warning
Iridescent / mesh-gradient backgrounds read as 2022 design template, not cyberpunk. ek-design hero atmosphere = flat dark + neon edge accents + optional scanline overlay.
pattern: emoji-as-icon error
Lucide SVG icons only. Emojis in UI chrome read as casual / consumer.
pattern: lorem-ipsum warning
Use real content or honest placeholders ('Task title here', 'Agent output appears below').
pattern: rounded-card-with-colored-left-border warning
Overused B2B SaaS trope.
pattern: decorative-neon-glow warning
Neon edge glow is reserved for focus, hover, and the agent_status_active badge. Decorative glow everywhere = cyberpunk costume, not brand.
